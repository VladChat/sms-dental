# Accessibility WCAG

Use this skill when building or reviewing UI, forms, and content.

Baseline requirements:
- Keyboard navigation for all interactive elements.
- Visible focus states.
- Sufficient text and UI contrast.
- Semantic headings and landmarks.
- Form labels, instructions, and error association.

Media and motion:
- Provide meaningful alt text for informative graphics.
- Avoid flashing or distracting motion.
- Respect reduced-motion preferences.

Interaction quality:
- Clear status feedback for loading and submission.
- Accessible modal and menu behavior.
- Large enough touch targets on mobile.

Dental SMS context:
- Keep critical actions (trial, demo, contact) accessible and unambiguous.
- Preserve trust by making policies and compliance text readable and reachable.
